# Planar Nexus v1.1 Roadmap

**Created**: March 12, 2026  
**Target Release**: Q2 2026 (April-June 2026)  
**Theme**: Customization & Connectivity

---

## Vision

v1.1 transforms Planar Nexus from a deck builder and playtester into a **creative platform** where users can design custom cards and seamlessly sync their collections across devices. This release focuses on user creativity and data portability.

### Strategic Goals

1. **Enable Creativity** — Custom card creation with AI artwork generation
2. **Improve Accessibility** — Cloud sync for decks across devices
3. **Enhance Engagement** — Achievement system to track milestones
4. **Strengthen Community** — Deck sharing and discovery features

---

## Major Features

### 1. Custom Card Creation Studio ✨ NEW

**Priority**: High  
**Effort**: 40-60 hours  
**Status**: Planned

#### Overview

A comprehensive WYSIWYG editor for creating custom cards with AI-generated artwork, custom abilities, and balanced stats.

#### Features

**Card Editor**:
- Visual card preview (real-time updates)
- Template system for different card types
- Custom ability text editor with syntax highlighting
- Mana cost designer with hybrid/phyrexian support
- Power/toughness/loyalty editors
- Custom frame colors and layouts

**AI Artwork Generation**:
- Integration with existing procedural art system
- Style presets (fantasy, sci-fi, abstract, etc.)
- Mood and atmosphere controls
- Iterative refinement (generate → adjust → regenerate)
- Upload custom artwork option

**Card Balance Tools**:
- AI-powered mana cost suggestions
- Power level assessment
- Format legality checker
- Comparison with similar cards

**Export Options**:
- Export as PNG/PDF (printable proxies)
- Export to deck builder (seamless integration)
- Share via URL or JSON
- Bulk export for custom sets

#### Technical Requirements

**Frontend**:
- New React components for card editor
- Canvas/SVG manipulation libraries
- Real-time preview rendering
- Drag-and-drop layout editor

**Backend**:
- Custom card database schema
- Card validation rules engine
- Image storage and caching
- Share URL generation

**AI Integration**:
- Enhanced procedural art generator
- Balance suggestion engine
- Ability text parser

#### User Stories

1. **As a custom card designer**, I want to create cards with custom abilities so that I can design my own fan sets.

2. **As a deck builder**, I want to use custom cards in my decks so that I can test unique strategies.

3. **As a playtester**, I want to generate AI artwork for my cards so that they look professional.

4. **As a content creator**, I want to share my custom cards via URL so that others can view and use them.

#### Success Metrics

- Users can create a custom card in <5 minutes
- AI artwork generation completes in <10 seconds
- 100+ custom cards created in first month
- 50+ custom decks using custom cards

### 2. Cloud Sync (Optional) ☁️ NEW

**Priority**: High  
**Effort**: 30-40 hours  
**Status**: Planned

#### Overview

Privacy-first optional cloud synchronization for decks, custom cards, and settings. Users maintain full control over their data.

#### Features

**Sync Options**:
- **Local Only** (default) — All data stays on device
- **Cloud Backup** — Encrypted backup to user's cloud storage
- **Full Sync** — Real-time sync across devices

**Supported Providers**:
- Google Drive
- Dropbox
- OneDrive
- iCloud Drive
- Self-hosted WebDAV

**Privacy Features**:
- End-to-end encryption (user holds keys)
- Zero-knowledge architecture
- Selective sync (choose what to sync)
- Local data export (full backup)

**Sync Management**:
- Conflict resolution (manual or automatic)
- Sync history and versioning
- Device management (view/remove devices)
- Bandwidth controls (sync frequency)

#### Technical Requirements

**Security**:
- AES-256 encryption at rest
- TLS 1.3 in transit
- User-controlled encryption keys
- No server-side decryption

**Architecture**:
- Local-first sync (works offline)
- CRDT-based conflict resolution
- Incremental sync (only changes)
- Background sync service

**Storage**:
- User's own cloud storage (no central server)
- Optional Planar Nexus cloud (paid tier)
- Efficient delta compression

#### User Stories

1. **As a multi-device user**, I want my decks to sync automatically so that I can build on my desktop and play on my laptop.

2. **As a privacy-conscious user**, I want end-to-end encryption so that only I can access my data.

3. **As a traveler**, I want offline access to my decks so that I can use the app without internet.

4. **As a backup-conscious user**, I want automatic cloud backups so that I don't lose my collection.

#### Success Metrics

- 60% of users enable cloud sync
- Sync completes in <5 seconds for typical deck
- Zero data loss incidents
- <1% sync conflict rate

## Decision Log

### Decisions Made

1. **Cloud Sync Architecture**: Local-first with user-controlled cloud storage
   - Date: 2026-03-12
   - Rationale: Privacy-first approach aligns with project values

2. **Custom Card Sharing**: URL-based sharing without central database
   - Date: 2026-03-12
   - Rationale: Reduces moderation burden, empowers users

3. **Achievement Rewards**: Cosmetic-only rewards (no gameplay advantages)
   - Date: 2026-03-12
   - Rationale: Maintains competitive integrity

1. **Mobile App Platform**: React Native vs Flutter
   - Deadline: 2026-04-01
   - Stakeholders: Core team, community input

2. **Cloud Sync Provider**: Build vs partner
   - Deadline: 2026-04-15
   - Stakeholders: Technical team, budget approval
