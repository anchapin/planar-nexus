# Planar Nexus v1.0.1 Patch Plan

**Created**: March 12, 2026  
**Target Release**: March 19-26, 2026 (1-2 weeks after v1.0.0)  
**Priority**: Medium — Stability and test reliability improvements

---

## Overview

v1.0.1 is a patch release focused on fixing medium-priority test infrastructure issues identified during the v1.0.0 QA process. This release does not include new features but improves test reliability, fixes test data seeding, and enhances E2E test stability.

### Patch Goals

1. **Fix E2E test flakiness** — Improve timing and selectors
2. **Seed card database test data** — Ensure consistent test environment
3. **Improve test selectors** — Make E2E tests more robust
4. **Address medium-priority issues** — Complete remaining test coverage gaps

### Success Criteria

- ✅ All E2E tests pass consistently (no flakiness)
- ✅ Test data seeding is reliable and repeatable
- ✅ Test selectors are robust across screen sizes
- ✅ Import/export round-trip tests pass
- ✅ Multiplayer lobby tests pass
- ✅ Coach component tests pass

## Issues to Address

### Medium Priority (5 Issues)

#### 1. Card Database Test Data Seeding

**Issue**: Test data for card database is not properly seeded, causing inconsistent test results.

**Impact**: Tests may fail when card database is empty or incomplete.

**Fix Plan**:
1. Create dedicated test data seeding utility
2. Add `beforeEach` hooks to ensure clean state
3. Implement test fixtures for common card sets
4. Add validation to verify seed data integrity

**Files to Modify**:
- `src/lib/__tests__/card-database.test.ts`
- `src/lib/__fixtures__/test-cards.ts` (new)
- `jest.setup.js`

**Estimated Effort**: 2-3 hours

---

#### 2. E2E Test Flakiness (Timing Issues)

**Issue**: Some E2E tests fail intermittently due to timing issues and race conditions.

**Impact**: CI/CD pipeline shows false negatives, requiring manual re-runs.

**Symptoms**:
- Tests fail on slow CI runners but pass locally
- Intermittent failures (10-15% failure rate)
- Timing-dependent assertions fail

**Fix Plan**:
1. Replace fixed `waitForTimeout` with proper `waitFor` assertions
2. Add network idle conditions where appropriate
3. Implement retry logic for flaky tests
4. Increase timeouts for CI environment only

**Files to Modify**:
- `e2e/*.spec.ts` (all E2E test files)
- `playwright.config.ts` (CI-specific timeouts)

**Estimated Effort**: 4-6 hours

---

#### 3. E2E Test Flakiness (Selector Issues)

**Issue**: Test selectors are brittle and break with minor UI changes.

**Impact**: Tests fail when component structure changes, even if functionality works.

**Current Problems**:
- Over-reliance on CSS selectors
- Text content matching is too strict
- No data-testid attributes on key elements

**Fix Plan**:
1. Add `data-testid` attributes to critical UI elements
2. Use role-based selectors where possible (Playwright best practices)
3. Implement page object model for complex flows
4. Create selector utilities for common patterns

**Files to Modify**:
- `src/app/**/*.tsx` (add data-testid)
- `e2e/**/*.spec.ts` (update selectors)
- `e2e/utils/selectors.ts` (new utility)

**Estimated Effort**: 6-8 hours

---

#### 4. Import/Export Round Trip E2E Test

**Issue**: No comprehensive E2E test for full import/export cycle.

**Impact**: Regression risk for import/export functionality.

**Test Coverage Needed**:
1. Export deck as JSON → Re-import → Verify identical
2. Export deck as text → Re-import → Verify identical
3. Export deck → Copy to clipboard → Import from clipboard
4. Export deck → Share URL → Import from URL

**Fix Plan**:
1. Create comprehensive import/export E2E test suite
2. Test all supported formats (JSON, text, MTGO, URL)
3. Verify deck integrity after round trip
4. Test edge cases (empty deck, maximum size deck)

**Files to Create**:
- `e2e/import-export-roundtrip.spec.ts`

**Estimated Effort**: 3-4 hours

---

#### 5. Multiplayer Lobby E2E Test Coverage

**Issue**: Multiplayer lobby flow lacks comprehensive E2E test coverage.

**Impact**: Regression risk for multiplayer functionality.

**Test Coverage Needed**:
1. Host game → Generate code → Join game → Start game
2. Join game with invalid code → Error handling
3. Host disconnect → Migration to new host
4. Spectator join → View game

**Fix Plan**:
1. Create multiplayer lobby E2E test suite
2. Mock WebRTC connections for testing
3. Test happy path and error scenarios
4. Add 2v2 and 3v3 team mode tests

**Files to Create**:
- `e2e/multiplayer-lobby.spec.ts`
- `e2e/utils/mock-webrtc.ts`

**Estimated Effort**: 6-8 hours
