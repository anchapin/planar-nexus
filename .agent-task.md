# Issue #761: Critical - Event sourcing not connected to state mutations

## Task

Wire EventSourcingGameState into all game-state.ts mutation functions. Every state mutation must emit events.

## Key Files

- `src/lib/game-state/event-sourcing.ts` - EventSourcingGameState class (already exists)
- `src/lib/game-state/game-state.ts` - Contains drawCard(), dealDamageToPlayer(), dealDamageToCreature(), moveCard(), addCounter(), removeCounter(), createToken()
- `src/lib/game-state/types.ts` - GameState, GameAction types

## Required Implementation

1. Create wrapper functions that emit events BEFORE mutation:
   - `drawCard(state, playerId)` → emit DRAW_CARD event, then mutate
   - `dealDamageToPlayer(state, playerId, damage, ...)` → emit DEAL_DAMAGE event
   - `dealDamageToCreature(state, cardId, damage, ...)` → emit DEAL_DAMAGE event
   - `moveCard(state, cardId, fromZone, toZone)` → emit ZONE_CHANGE event
   - `addCounter(state, cardId, counterType, count)` → emit ADD_COUNTER event
   - `removeCounter(state, cardId, counterType, count)` → emit REMOVE_COUNTER event
   - `createToken(state, ...)` → emit CREATE_TOKEN event

2. Use `withEventEmission()` helper from event-sourcing.ts or create events directly

3. Ensure state hash verification after each mutation

## Acceptance Criteria

- All state mutations emit events through EventSourcingGameState
- Events can be replayed to reconstruct game state
- State hash verification works after each mutation

Run `npm test -- --grep="event.*sourcing"` to verify
