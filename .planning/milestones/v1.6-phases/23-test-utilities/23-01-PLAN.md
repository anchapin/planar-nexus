# Plan: Test Utilities Library

**Phase:** 23  
**Goal:** Create reusable test utilities library for consistent, maintainable tests across the codebase  
**Created:** 2026-03-19

---

## Objective Summary

Build a comprehensive `@/test-utils` library that provides:

1. Standardized React component testing helpers
2. Mock implementations for browser APIs and external services
3. Test data factories for consistent test data generation
4. Documentation for test utility usage

This enables faster test development, reduces test boilerplate, and ensures consistent testing patterns.

---

## Tasks

### Wave 1: Directory Structure & Core Setup

#### Task 1.1: Create test-utils directory structure

- **Action:** Create `src/test-utils/` directory with subdirectories:
  - `src/test-utils/` (root)
  - `src/test-utils/__mocks__/` (mock implementations)
  - `src/test-utils/factories/` (data factories)
  - `src/test-utils/helpers/` (utility helpers)
- **Verify:** Directory structure exists with `ls -la src/test-utils/`
- **Done:** When `src/test-utils/` directory is created with all subdirectories

#### Task 1.2: Create test-utils index file

- **Action:** Create `src/test-utils/index.ts` that exports all utilities
- **Verify:** File exports all test utilities
- **Done:** When index.ts exists and exports all modules

### Wave 2: React Component Test Helpers

#### Task 2.1: Build render helper with providers

- **Action:** Create `src/test-utils/helpers/render.tsx`:
  - Wrap component with all providers (ThemeProvider, QueryProvider, etc.)
  - Support custom provider overrides
  - Return { rerender, unmount } from render
- **Verify:** Run existing component test with new render helper
- **Done:** When `renderWithProviders()` works for existing component tests

#### Task 2.2: Create screen query helpers

- **Action:** Create `src/test-utils/helpers/queries.ts`:
  - Export common queries (getByCardName, getByDeckTitle, etc.)
  - Wrap Testing Library queries with custom prefixes
  - Add async query variants for loading states
- **Verify:** Helpers work in component test
- **Done:** When queries can find custom components (Card, Deck, etc.)

#### Task 2.3: Create user event setup

- **Action:** Create `src/test-utils/helpers/user-event.ts`:
  - Pre-configured userEvent.setup() for consistent interaction testing
  - Custom interaction helpers (clickCard, addToDeck, etc.)
- **Verify:** user-event works in interaction test
- **Done:** When custom interaction helpers work correctly

### Wave 3: Mock Implementations

#### Task 3.1: Create fetch mock

- **Action:** Create `src/test-utils/__mocks__/fetch.ts`:
  - Mock successful Scryfall API responses
  - Support mockResolvedValueOnce for different responses
  - Include card search, card by ID, set list responses
- **Verify:** Fetch mock resolves with mock data
- **Done:** When fetch mock returns test data

#### Task 3.2: Create storage mocks

- **Action:** Create `src/test-utils/__mocks__/storage.ts`:
  - Mock localStorage with getItem, setItem, removeItem, clear
  - Mock sessionStorage with same API
  - Support getAllKeys() for iteration
- **Verify:** Storage mock persists data within test
- **Done:** When localStorage/sessionStorage work in tests

#### Task 3.3: Create router mock

- **Action:** Create `src/test-utils/__mocks__/next-router.tsx`:
  - Mock Next.js useRouter hook
  - Support push, replace, back navigation
  - Include query params and pathname
- **Verify:** Router mock navigates correctly
- **Done:** When useRouter works in component tests

### Wave 4: Test Data Factories

#### Task 4.1: Create card factory

- **Action:** Create `src/test-utils/factories/card.ts`:
  - Factory function: `createCard(overrides?)`
  - Default valid card with all required Scryfall fields
  - Support override for name, type_line, cmc, colors, etc.
  - Include variant builders: createLand(), createCreature(), createSpell()
- **Verify:** Factory generates valid card object
- **Done:** When `createCard()` returns valid ScryfallCard

#### Task 4.2: Create deck factory

- **Action:** Create `src/test-utils/factories/deck.ts`:
  - Factory function: `createDeck(overrides?)`
  - Default valid deck with commander format
  - Support override for name, format, cards array
  - Include variants: createLimitedDeck(), createStandardDeck()
- **Verify:** Factory generates valid deck object
- **Done:** When `createDeck()` returns valid SavedDeck

#### Task 4.3: Create game state factory

- **Action:** Create `src/test-utils/factories/game-state.ts`:
  - Factory for AI game state objects
  - Support different game phases (mulligan, main, combat)
  - Include player/opponent hands, libraries, battlefield
- **Verify:** Factory generates valid game state
- **Done:** When `createGameState()` returns valid game state
