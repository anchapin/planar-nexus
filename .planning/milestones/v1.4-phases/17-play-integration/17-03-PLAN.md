# Plan 17-03: Post-Game Return to Limited Session

## Overview
After a single-player game ends, return the user to their draft or sealed session to continue building or modify their deck.

## Requirements
- **LPLY-03**: Post-game returns user to limited session

## Implementation Steps

### Step 1: Track Session Context
- [ ] Extend limited session state to include:
  - `sessionType`: 'draft' | 'sealed'
  - `sessionId`: unique identifier for the session
  - `returnUrl`: path to return to
- [ ] Store this context when launching the game from deck builder

### Step 2: Add Return Button to Game End Screen
- [ ] Update single-player game end state/redux
- [ ] Show "Return to Draft" or "Return to Sealed" button prominently
- [ ] Include result summary (wins/losses if tracking)

### Step 3: Implement Return Navigation
- [ ] Create navigation function that restores session context
- [ ] Return to draft: `/draft?sessionId=xyz` (resume picking)
- [ ] Return to sealed: `/sealed?sessionId=xyz` (resume pool building)
- [ ] Return to deck builder: `/limited-deck-builder?sessionId=xyz` (continue editing)

### Step 4: Preserve State After Game
- [ ] Ensure pool cards are not lost after game
- [ ] Allow user to modify deck after game if desired
- [ ] Handle case where user wants to start a new game with same deck

### Step 5: Handle Game Abandonment
- [ ] If user closes browser mid-game, session should still be recoverable
- [ ] Provide way to resume from session list if not auto-returned

## Verification Criteria
- [ ] After game ends, user sees "Return to Session" button
- [ ] Clicking return navigates back to correct session type
- [ ] Pool and deck state is preserved after return
- [ ] User can play another game with same or modified deck

## Dependencies
- Plan 17-01 (game launch with session context)
- Session management in draft and sealed modules
