# Plan 17-02: Limited Deck Format Validation

## Overview
Implement validation to ensure limited decks meet format requirements (40 cards minimum, 4-copy limit per card) before allowing game launch.

## Requirements
- **LPLY-02**: Limited deck format validated before AI game starts

## Implementation Steps

### Step 1: Create Deck Validation Utility
- [ ] Create `/src/lib/limited-deck-validation.ts`
- [ ] Export `validateLimitedDeck(deck: DeckCard[]): ValidationResult` function
- [ ] Validation rules:
  - Minimum 40 cards
  - Maximum 4 copies of any single card
  - All cards must be from the limited pool (session-scoped)
- [ ] Return detailed validation messages for each error

### Step 2: Integrate Validation into Limited Deck Builder
- [ ] Import validation utility into `/src/app/(app)/limited-deck-builder/page.tsx`
- [ ] Run validation on deck changes
- [ ] Display validation status (valid/invalid) with error messages
- [ ] Disable "Play vs AI" button when deck is invalid

### Step 3: Validate Before Game Launch
- [ ] Re-validate deck in the transfer/logic before launching game
- [ ] Show error and prevent launch if deck becomes invalid
- [ ] Handle edge case of deck becoming invalid between builder and launch

### Step 4: UI Feedback for Validation
- [ ] Show card count (e.g., "35/40 cards")
- [ ] Highlight cards exceeding 4-copy limit in red
- [ ] Provide clear "Deck Ready" indicator when valid

## Verification Criteria
- [ ] Deck with 39 cards shows "Need 1 more card" error
- [ ] Deck with 5 copies of a card shows "Exceeds 4-copy limit" error
- [ ] Valid 40+ card deck shows "Deck Ready" indicator
- [ ] Play button is disabled until deck is valid

## Dependencies
- Plan 17-01 (the play button)
- Limited deck builder must store deck as `DeckCard[]` format
