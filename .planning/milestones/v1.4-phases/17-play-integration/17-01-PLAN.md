# Plan 17-01: Launch AI Game with Built Limited Deck

## Overview
Add functionality to launch an AI opponent game directly from the limited deck builder with the user's built limited deck.

## Requirements
- **LPLY-01**: User can launch AI opponent game with built limited deck

## Implementation Steps

### Step 1: Add Play Button to Limited Deck Builder UI
- [ ] Open `/src/app/(app)/limited-deck-builder/page.tsx`
- [ ] Add "Play vs AI" button that appears when deck meets minimum requirements
- [ ] Button should be disabled and show tooltip when deck is invalid
- [ ] Style button consistently with existing action buttons

### Step 2: Create Deck Transfer Logic
- [ ] Add server action or utility function to prepare limited deck for single-player
- [ ] Convert limited deck format to single-player compatible format
- [ ] Generate random AI opponent deck (or select from prebuilt decks)
- [ ] Store session context for return navigation

### Step 3: Integrate with Single Player Router
- [ ] Update `/src/app/(app)/single-player/page.tsx` to accept limited deck parameter
- [ ] Detect deck type and initialize game appropriately
- [ ] Pass session ID for return navigation

### Step 4: Handle Edge Cases
- [ ] What if user has no valid deck? (show error message)
- [ ] What if single-player route fails to load? (show error, return to builder)
- [ ] Session timeout handling

## Verification Criteria
- [ ] User can click "Play vs AI" from limited deck builder when deck has 40+ cards
- [ ] Game launches with user's deck against AI opponent
- [ ] Session context is preserved for return navigation

## Dependencies
- Phase 16 (AI Neighbors) should be complete for passing mechanics
- Limited deck builder must exist and be functional
- Single-player game must be functional
