# Plan 1.1: Card Database Expansion

## Objective
Expand card database from 20 cards to 500+ cards with efficient storage, caching, and retrieval.

## Why This Matters
- Current 20-card database is insufficient for meaningful deck building
- AI opponent needs 200+ card pool for diverse decks
- Users need variety for deck experimentation
- Foundation for all other features

### Task 1.1.4: Create Card Selection Strategy
**Type**: decision  
**Duration**: ~10 min (human decision)

**Decision**: Which 500 cards to import for v1.0?

**Options**:

**Option A: Curated Balanced Pool**
- 50 cards per color (WUBRG)
- 100 artifacts
- 100 lands (basic + dual)
- 50 multicolor
- Pros: Balanced for all deck types, good AI pool
- Cons: May miss popular cards

**Option B: Popular Commander Cards**
- Top 500 most-played Commander cards (EDHREC data)
- Pros: Users recognize cards, meta-relevant
- Cons: May be color-imbalanced, expensive API calls

**Option C: All Cards by CMC Distribution**
- 50 cards at each CMC 0-8+ (450 cards)
- 50 lands
- Pros: Ensures curve-building options
- Cons: May include useless cards

**Recommendation**: Option A (Curated Balanced Pool) for v1.0

**Resume Signal**: "Use Option A" / "Use Option B" / "Use Option C"
