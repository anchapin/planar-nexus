# Plan 1.6: Build Pipeline

## Objective
Fix TypeScript errors, enable strict checks, and ensure clean builds for production release.

## Why This Matters
- Build errors suppressed in `next.config.ts` mask real issues
- TypeScript `any` types reduce code quality
- Broken tests prevent regression detection
- Required for confident v1.0 release

---

## Tasks

### Task 1.6.1: Audit Current Build Configuration
**Type**: research  
**Duration**: ~20 min

**Actions**:
1. Read `next.config.ts` - identify suppressed errors
2. Read `tsconfig.json` - check strictness settings
3. Run `npm run build` - count actual errors
4. Run `npm run lint` - identify lint issues
5. Run `npm test` - check test status

**Deliverable**: Build health report with error counts

---

### Task 1.6.2: Fix TypeScript `any` Types
**Type**: auto  
**Duration**: ~60 min

**Actions**:
1. Search for `: any` in codebase
2. Replace with proper types:
   - Function return types
   - Event handler types
   - API response types
   - Component prop types
3. Enable `noImplicitAny` in tsconfig
4. Fix any new errors that appear

**Verification**:
```bash
npm run type-check
# Should show 0 errors
```

---

### Task 1.6.3: Enable Strict TypeScript Checks
**Type**: auto  
**Duration**: ~30 min

**Actions**:
1. Update `tsconfig.json`:
```json
{
  "compilerOptions": {
    "strict": true,
    "noImplicitAny": true,
    "noImplicitReturns": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true
  }
}
```
2. Fix errors that appear from strict mode
3. Focus on critical paths first (game engine, deck builder)

**Verification**:
- TypeScript compiles with zero errors
- Strict mode enabled

---

### Task 1.6.4: Fix Next.js Build Errors
**Type**: auto  
**Duration**: ~45 min

**Actions**:
1. Remove error suppression from `next.config.ts`:
```typescript
// REMOVE THIS:
// eslint-disable-next-line @typescript-eslint/no-unused-vars
// Experimental features causing build errors

// FIX THE ROOT CAUSE instead
```
2. Fix any ESLint errors that were suppressed
3. Run `npm run build` until clean

**Verification**:
```bash
npm run build
# Should complete without errors
```

### Task 1.6.7: Add Pre-commit Hooks
**Type**: auto  
**Duration**: ~20 min

**Actions**:
1. Install Husky: `npx husky-init`
2. Create pre-commit hook:
```bash
#!/bin/sh
npm run type-check
npm run lint
npm test -- --passWithNoTests
```
3. Test hook by making a commit

**Verification**:
- Commits blocked if type-check fails
- Commits blocked if lint fails
