# v1.6 Roadmap: QA/QC Infrastructure

## Goal
Establish comprehensive quality engineering infrastructure with pre-commit hooks, test strategies, and coverage reporting.

## Phase 1: Pre-commit Hooks & Coverage Foundation

### Phase 21: Pre-commit Hooks Setup
**Plans:**
- [P1] Install and configure Husky git hooks
- [P2] Set up lint-staged for staged file processing
- [P3] Configure pre-commit ESLint check
- [P4] Add Prettier formatting check to pre-commit
- [P5] Add TypeScript type check to pre-commit
- [P6] Test pre-commit hooks locally

### Phase 22: Coverage Infrastructure
**Plans:**
- [P1] Configure Jest coverage thresholds in jest.config.js
- [P2] Add coverage gate to CI pipeline
- [P3] Configure multi-format coverage reports (lcov, html)
- [P4] Set up coverage badge configuration

## Phase 2: Test Utilities & Integration Tests

### Phase 23: Test Utilities Library
**Plans:**
- [P1] Create @/test-utils directory structure
- [P2] Build React component test helpers
- [P3] Create mock implementations (fetch, localStorage, sessionStorage)
- [P4] Build test data factories for card data, decks
- [P5] Document test utilities usage

### Phase 24: Integration Test Setup
**Plans:**
- [P1] Install and configure MSW (Mock Service Worker)
- [P2] Create MSW handlers for Scryfall API
- [P3] Build integration test patterns for server actions
- [P4] Add integration tests for deck CRUD operations
- [P5] Add integration tests for card search functionality

## Phase 3: CI Enhancement & Documentation

### Phase 25: CI Quality Gates
**Plans:**
- [P1] Add coverage threshold gate to CI
- [P2] Configure commit message linting
- [P3] Add npm audit security scanning to CI
- [P4] Add branch protection rule recommendations

### Phase 26: Test Documentation
**Plans:**
