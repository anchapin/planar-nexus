# Phase 19 Plan 3: Game Phase Strategy Guides

**Phase**: 19 - Matchup & Strategy Guides  
**Subsystem**: AI Strategy  
**Priority**: Medium  
**Dependencies**: Phase 18 (completed), Plan 19-02

---

## Context

Users need in-game strategic guidance based on deck archetype and game phase. This plan implements opening hand evaluation, mid-game priorities, late-game advice, and combat phase guidance.

## Requirements Covered

| ID | Requirement | Description |
|----|-------------|-------------|
| PHASE-01 | Opening hand tips | User can get opening hand evaluation tips |
| PHASE-02 | Mid-game priorities | User can see mid-game strategic priorities by deck type |
| PHASE-03 | Late-game advice | User can get late-game top-deck scenarios advice |
| PHASE-04 | Combat phase guidance | User can receive combat phase decision guidance |

## Implementation Plan

### Task 1: Create GamePhaseStrategy Data Types

**File**: `src/lib/game-phase-strategy.ts`

- Create `GamePhaseStrategy` interface:
  - `archetype: ArchetypeCategory` - deck type
  - `opening: PhaseStrategy`
  - `midGame: PhaseStrategy`
  - `lateGame: PhaseStrategy`
  - `combat: CombatTips`

- Create `PhaseStrategy`:
  - `priorities: string[]` - top priorities
  - `commonMistakes: string[]` - what to avoid
  - `idealScenario: string` - what you're aiming for
  - `redFlags: string[]` - warning signs

- Create `CombatTips`:
  - `attacking: CombatGuidance`
  - `defending: CombatGuidance`
  - `general: string[]`

- Create `CombatGuidance`:
  - `whenTo: string[]` - when to attack/block
  - `whenNotTo: string[]` - when to hold back
  - `calculations: string[]` - math to consider

- Create mock data for all archetypes:
  - Aggro, Control, Midrange, Combo, Tempo strategies
  - Format-specific nuances (Commander = higher life, etc.)

- Helper functions:
  - `getGamePhaseStrategy(archetype, format)`
  - `getCombatTips(archetype)`
  - `evaluateHand(hand, deckArchetype)` - returns evaluation

### Task 2: Create PhaseStrategyCard Component

**File**: `src/components/meta/PhaseStrategyCard.tsx`

- Props: `PhaseStrategy`, phase name
- Display:
  - Phase name header with icon
  - Priorities list (numbered)
  - Common mistakes (warning style)
  - Ideal scenario callout
  - Red flags section
- Uses Card, List, Badge components

### Task 3: Create CombatGuidance Component

**File**: `src/components/meta/CombatGuidance.tsx`

- Props: `CombatTips`
- Tabs: Attacking | Defending | General
- Display:
  - When to / When not to lists
  - Calculations to consider
- Uses Tabs component from shadcn

### Task 4: Create HandEvaluator Component

**File**: `src/components/meta/HandEvaluator.tsx`

- Input: Manual card entry or paste from deck
- Display:
  - Hand quality rating (Excellent/Good/Fair/Poor)
  - Key factors analysis
  - Suggestions for improvement
- Uses Card, Input, Button components

### Task 5: Create GamePhaseStrategyPage

**File**: `src/app/(app)/strategy/page.tsx`

- Full page for game phase strategies
- Archetype selector at top
- Tabbed view: Opening | Mid-Game | Late-Game | Combat
- HandEvaluator tool as bonus feature
- Add to sidebar navigation

### Task 6: Update Strategy Service with Deck Analysis

**File**: `src/lib/game-phase-strategy.ts`

- Extend `evaluateHand` to:
  - Check mana curve in hand
  - Evaluate color distribution
  - Assess curve vs. deck curve
  - Identify power Turn/Turns
