# Plan 20-01: Custom Sideboard Plans

**Phase**: 20  
**Plan**: 20-01  
**Objective**: Enable users to save and manage custom sideboard plans

---

## Requirements

- **SIDE-03**: User can save custom sideboard plans

## Success Criteria

- [ ] Users can create new custom sideboard plans
- [ ] Users can edit existing sideboard plans
- [ ] Sideboard plans persist in localStorage
- [ ] Users can delete custom plans
- [ ] Plans show in/out recommendations with card counts

---

## Tasks

### Task 1: Create Sideboard Plans Data Types & Storage
**Action**: Create `src/lib/sideboard-plans.ts`
- TypeScript interfaces for SavedSideboardPlan
- localStorage helper functions (save, load, delete, list)
- Validation for plan data

**Verify**: File created with proper types and functions

**Done**: Types and storage layer complete

---

### Task 2: Create SideboardPlanCard Component
**Action**: Create `src/components/meta/sideboard/SideboardPlanCard.tsx`
- Display plan name, archetype, format
- Show in/out card counts
- Edit and delete buttons
- Click to view details

**Verify**: Component renders with all fields

**Done**: Card component complete

---

### Task 3: Create SideboardPlanEditor Dialog
**Action**: Create `src/components/meta/sideboard/SideboardPlanEditor.tsx`
- Form for plan name
- Format selector (Standard/Modern/Commander)
- Player archetype dropdown
- In/out card lists with add/remove
- Save/Cancel buttons

**Verify**: Dialog opens and closes properly

**Done**: Editor dialog functional

---

### Task 4: Integrate with AntiMetaRecommendations
**Action**: Modify `src/components/meta/AntiMetaRecommendations.tsx`
- Add "Save as Custom Plan" button in sideboard tab
- Open SideboardPlanEditor with pre-filled data
- After save, show custom plans list

**Verify**: Can save sideboard recommendations as custom plan

**Done**: Integration complete

---

### Task 5: Create My Sideboard Plans Page
**Action**: Create `src/app/(app)/sideboards/page.tsx`
- List all saved custom plans
- Filter by format
- Card grid of SideboardPlanCard components
- Empty state with CTA

**Verify**: Page loads and displays saved plans

**Done**: Page functional
