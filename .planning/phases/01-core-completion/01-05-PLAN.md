# Plan 1.5: Single Player Complete

## Objective
Remove "Prototype" status and deliver a complete, polished single-player experience from deck selection through game result.

## Why This Matters
- This is the v1.0 success criterion feature
- User's first experience with the app
- Must work flawlessly within 10 minutes of install
- Showcases all core features working together

---

### Task 1.5.3: Polish Deck Selection Flow
**Type**: auto  
**Duration**: ~30 min

**Actions**:
1. Show user's saved decks prominently
2. Add deck preview (mana curve, colors, creature count)
3. Quick-select recent decks
4. "Create New Deck" button → navigate to deck builder
5. Import deck option inline
6. Show format legality indicator

**Verification**:
- User can select deck in ≤2 clicks
- Deck info visible at a glance

---

### Task 1.5.4: Polish AI Opponent Configuration
**Type**: auto  
**Duration**: ~30 min

**Actions**:
1. Create clean AI config UI:
   - Difficulty selector (Easy, Medium, Hard, Expert)
   - Archetype selector (Aggro, Control, Midrange, etc.)
   - Theme selector (Burn, Zombies, Dragons, etc.)
   - "Random" option for variety
2. Show recommended configs for learning
3. Save user's recent configs
4. Preview AI deck stats (avg CMC, colors)

**Verification**:
- Config is intuitive
- User understands what each option means

---

### Task 1.5.5: Implement Game Start Flow
**Type**: auto  
**Duration**: ~30 min

**Actions**:
1. "Start Game" button triggers:
   - Validate user deck (60+ cards for Standard, 100 for Commander)
   - Generate AI deck based on config
   - Initialize GameState
   - Navigate to game board
2. Add loading screen during initialization:
   - "Shuffling decks..."
   - "Determining first player..."
   - "Drawing opening hands..."
3. Handle errors gracefully:
   - Invalid deck → show error, stay on page
   - Initialization failure → retry option

**Verification**:
- Game starts within 2 seconds
- Clear feedback during loading
- Errors are recoverable

---

### Task 1.5.6: Implement Game Result Tracking
**Type**: auto  
**Duration**: ~30 min

**Actions**:
1. Detect game end conditions:
   - Player life = 0
   - Player cannot draw (decked out)
   - Concede
   - Timer (for future timed modes)
2. Show result screen:
   - Victory/Defeat/Draw
   - Game stats (turns played, life total, creatures killed)
   - "Play Again" button (same config)
   - "New Game" button (back to config)
   - "Change Deck" button (back to deck select)
3. Save game result to history:
   - Date, opponent, result, turns
   - Accessible from dashboard

**Verification**:
- Game ends correctly
- Result screen shows accurate info
- History is saved

---

### Task 1.5.7: Add First-Time User Onboarding
**Type**: auto  
**Duration**: ~30 min

**Actions**:
1. Detect first-time users (no saved decks, no games played)
2. Show welcome tooltip tour:
   - "Select a deck or create one"
   - "Configure your AI opponent"
   - "Click Start Game to playtest"
3. Provide pre-built starter decks:
   - "Aggro Burn" (simple, fast)
   - "Control Wizard" (learn stacking)
   - "Creature Ramp" (balanced)
4. Add "How to Play" link to rules

**Verification**:
- First-time users see helpful guidance
- Starter decks are playable and instructive

---

### Task 1.5.8: Add Game Speed Controls
**Type**: auto  
**Duration**: ~20 min

**Actions**:
1. Add speed toggle in game:
   - Normal (default pacing)
   - Fast (reduced delays)
   - Instant (no delays, for testing)
2. Persist user preference
3. Show current speed indicator

**Verification**:
- Speed changes affect AI pacing
- Preference saved across sessions

---

### Task 1.5.9: End-to-End Testing
**Type**: checkpoint:human-verify  
**Duration**: ~15 min

**What Built**: Complete single-player experience

**How to Verify**:
1. Run `npm run dev`
2. Open in fresh browser (simulate first-time user)
3. Time the full flow:
   - Land on single-player page
   - Select or create deck
   - Configure AI opponent
   - Start game
   - Play until AI takes a turn
   - Complete game (win or lose)
   - See result screen
4. Total time should be ≤10 minutes for first-time user
5. Verify no "Prototype" labels anywhere

**Resume Signal**: "Single player verified in X minutes" or describe issues
