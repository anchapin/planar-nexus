# Plan 1.4: AI Gameplay Loop

## Objective
Connect the AI decision engine to the game engine so AI can play complete games autonomously.

## Why This Matters
- AI opponent exists but doesn't actually play cards
- Critical for playtesting feature (REQ-5)
- Blocks v1.0 success criterion (10-minute playtest)
- Most critical gap in the codebase

### Task 1.4.2: Create AI Action Executor
**Type**: auto  
**Duration**: ~45 min

**Actions**:
1. Create `src/ai/ai-action-executor.ts`:
```typescript
interface AIAction {
  type: 'play_land' | 'cast_spell' | 'attack' | 'block' | 'activate_ability' | 'pass_priority';
  targetId?: string;
```

### Task 1.4.3: Implement AI Turn Loop
**Type**: auto  
**Duration**: ~60 min

**Actions**:
1. Create `src/ai/ai-turn-loop.ts`:
```typescript
export async function runAITurn(
  gameState: GameState,
  aiPlayerId: string,
  difficulty: 'easy' | 'medium' | 'hard' | 'expert'
): Promise<void>;
```
2. Implement phase-by-phase AI behavior:
   - **Untap**: Auto-untap all permanents
   - **Draw**: Auto-draw for turn
   - **Main 1**: Play lands, cast spells, use abilities
   - **Pre-combat**: Additional spells
   - **Combat**: Declare attackers, resolve combat
   - **Post-combat**: Additional spells
   - **End**: End turn
3. Add difficulty-based decision quality:
   - Easy: Random valid moves, suboptimal plays
   - Medium: Basic heuristics, decent plays
   - Hard: Advanced evaluation, strong plays
   - Expert: Deep analysis, optimal plays
4. Add delays for natural pacing (500ms-2s per action)

**Verification**:
- AI completes full turn without hanging
- Actions are valid per game rules
- Difficulty affects play quality

## Risks

| Risk | Mitigation |
|------|------------|
| AI makes illegal moves | Validate all actions before execution |
| AI hangs on complex decisions | Add timeout, fall back to pass |
| Performance issues with evaluation | Cache evaluations, limit depth |
| AI too easy/hard | Tune difficulty configs iteratively |
