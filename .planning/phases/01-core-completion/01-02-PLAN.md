# Plan 1.2: Game Engine Integration

## Objective
Connect the GameState engine to all UI components, replacing mock data with real game state.

## Why This Matters
- Game engine exists but UI uses mock data
- Single player shows "Prototype" because it's not wired up
- Cannot playtest without real game state
- Foundation for AI gameplay

### Task 1.2.3: Create Game State React Hook
**Type**: auto  
**Duration**: ~45 min

**Actions**:
1. Create `src/hooks/use-game-state.ts`:
```typescript
interface UseGameStateReturn {
  gameState: GameState | null;
  initializeGame: (config: GameConfig) => Promise<void>;
  dispatchAction: (action: GameAction) => Promise<void>;
  isLoading: boolean;
  error: Error | null;
}

export function useGameState(): UseGameStateReturn;
```
2. Implement IndexedDB persistence for saved games
3. Add subscription pattern for real-time updates
4. Handle multiplayer sync (future-proof)

**Verification**:
- Hook returns null before game starts
- Hook returns GameState after initialization
- State updates propagate to all subscribers
