<task type="checkpoint:human-verify" gate="blocking">
  <name>Verify Sealed Session Flow</name>
  <what-built>
    Complete sealed mode foundation:
    - Set browser with sorting and selection
    - Sealed pool generation (6 packs, 84 cards)
    - Pool filtering (color, type, CMC)
    - Limited deck builder with 40-card/4-copy validation
    - Pool isolation and session persistence
  </what-built>
  <how-to-verify>
    **Test 1: Set Browser (SET-01, SET-02, SET-03)**
    1. Navigate to http://localhost:9002/set-browser
    2. Verify sets load and display (may take a moment on first load - Scryfall API)
    3. Verify sets are sorted by release date (newest first)
    4. Click "Sort by Name" - verify alphabetical sort
    5. Click any set - verify modal shows set details with card count
    6. Click "Start Sealed" - verify navigation to /sealed page

    **Test 2: Sealed Pool (SEAL-01, SEAL-02)**
    1. On sealed page, verify session creation
    2. Verify pool shows 84 cards (6 packs × 14 cards)
    3. Verify all cards are immediately visible (no face-down cards)
    4. Note: May need to wait for card database to load set cards

    **Test 3: Pool Filtering (SEAL-03)**
    1. Click color filter buttons (e.g., "Blue")
    2. Verify only blue cards display
    3. Click type filter - select "Creature"
    4. Verify only blue creatures display
    5. Use CMC filter - verify filtered results
    6. Clear filters - verify all cards return

    **Test 4: Limited Deck Builder (LBld-01 to LBld-06)**
    1. Click "Build Deck" button
    2. Verify "Limited" mode is active (pool cards only)
    3. Try to switch to "Collection" mode - should be restricted
    4. Add cards to deck - verify card count increases
    5. Add 5+ copies of same card - verify error message appears
    6. Build deck to 39 cards - verify error about 40-card minimum
    7. Add one more card to reach 40 - verify no errors
    8. Verify no sideboard section visible
    9. Refresh page - verify deck and pool persist

    **Test 5: Pool Isolation (ISOL-01, ISOL-02, ISOL-03)**
    1. Navigate to regular deck builder (/deck-builder)
    2. Search for a card from your sealed pool
    3. Verify pool cards do NOT appear in collection search
    4. Open browser DevTools -> Application -> IndexedDB
    5. Verify "limited-sessions" store exists (separate from "decks")
    6. Verify each session has unique UUID id
  </how-to-verify>
  <resume-signal>Type "approved" if all tests pass, or describe issues found</resume-signal>
</task>
