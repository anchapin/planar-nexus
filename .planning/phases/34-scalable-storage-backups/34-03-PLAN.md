IDBKeyRange.upperBound(upper, open?)   // <= upper (or < if open=true)

<task type="auto">
  <name>Task 1: Audit Schema and Add Missing Indexes to PlanarNexusStorage</name>
  <files>src/lib/indexeddb-storage.ts</files>
  <action>
    **Audit findings** (document in code comments):

    PlanarNexusStorage gaps identified:
    - `game-history`: Missing compound index `date_result` for "show losses in date range" queries
    - `game-history`: Missing `opponent` index for "games vs specific opponent" queries
    - `saved-games`: `savedAt` is a number (Unix ms) — range queries work but need IDBKeyRange.bound()
    - `usage-tracking`: Missing compound `provider_timestamp` for "usage by provider in date range"
    - `decks`: `updatedAt` index exists ✅ (needed for STOR-01 incremental backup)

    **Changes to make:**

    1. Bump `DEFAULT_STORAGE_CONFIG.version` from `2` to `3`

    2. In `onupgradeneeded`, add `oldVersion` guard so existing stores are not recreated:
    ```typescript
    request.onupgradeneeded = (event) => {
      const db = (event.target as IDBOpenDBRequest).result;
      const oldVersion = event.oldVersion;

      // Version 1→2: create all stores (existing logic)
      if (oldVersion < 2) {
        // ... existing store creation code ...
      }

      // Version 2→3: add new indexes to existing stores
      if (oldVersion < 3) {
        const transaction = (event.target as IDBOpenDBRequest).transaction!;

        // Add compound index to game-history
        const gameHistoryStore = transaction.objectStore('game-history');
        if (!gameHistoryStore.indexNames.contains('date_result')) {
          gameHistoryStore.createIndex('date_result', ['date', 'result'], { unique: false });
        }
        if (!gameHistoryStore.indexNames.contains('opponent')) {
          gameHistoryStore.createIndex('opponent', 'opponent', { unique: false });
        }

        // Add compound index to usage-tracking
        const usageStore = transaction.objectStore('usage-tracking');
        if (!usageStore.indexNames.contains('provider_timestamp')) {
          usageStore.createIndex('provider_timestamp', ['provider', 'timestamp'], { unique: false });
        }
      }
    };
    ```

    3. Add `queryByDateRange<T>()` helper method to IndexedDBStorage:
    ```typescript
    async queryByDateRange<T>(
      storeName: string,
      indexName: string,
      from: number | string,
      to: number | string
    ): Promise<T[]> {
      return this.queryByIndex<T>(
        storeName,
        indexName,
        IDBKeyRange.bound(from, to, false, false)
      );
    }
    ```

    **IMPORTANT:** The `onupgradeneeded` refactor must preserve all existing store creation logic. Wrap the existing store creation in `if (oldVersion < 2)` block. Do not change any existing index names or store configurations.

  </action>
