# Plan 2.1: GameState Format Unification

## Objective
Unify Engine and AI GameState formats to enable full AI decision-making with real game state.

## Why This Matters
- Current AI uses simplified GameState format that differs from engine
- AI turn loop uses "attack with all creatures" instead of real decisions
- Full AI integration requires format bridge
- Enables Plans 2.2-2.5 to work with real game state

### Task 2.1.2: Design Unified GameState Interface
**Type**: decision
**Duration**: ~30 min

**Decision**: Where should unified GameState live?

**Options**:

**Option A: `src/lib/game-state/types.ts`**
- Pros: Engine is source of truth, AI adapts
- Cons: AI-specific fields may clutter engine types

**Option B: `src/lib/game-state/unified-types.ts`**
- Pros: Clean separation, both adapt to new file
- Cons: Third file to maintain

**Option C: Keep separate, add converters**
- Pros: No breaking changes to existing code
- Cons: Ongoing maintenance of two formats

**Recommendation**: Option A — Engine types are canonical, AI adapts. Add AI-specific fields as optional.

**Resume Signal**: "Use Option A" / "Use Option B" / "Use Option C"

### Task 2.1.3: Create Conversion Functions
**Type**: auto
**Duration**: ~90 min

**Actions**:
1. Create `src/lib/game-state/serialization.ts`:
```typescript
// Engine → AI format
export function engineToAIState(engineState: EngineGameState): AIGameState;

// AI → Engine format (for AI action results)
export function aiToEngineState(aiState: AIGameState, baseEngineState: EngineGameState): EngineGameState;

// Engine → Unified format
export function engineToUnified(engineState: EngineGameState): UnifiedGameState;

// Unified → Engine format
export function unifiedToEngine(unifiedState: UnifiedGameState): EngineGameState;
```

2. Handle all field mappings:
