# Plan 2.5: AI Opponent Difficulty Tuning

## Objective
Tune AI difficulty levels so each feels distinct and provides appropriate challenge (REQ-5.1).

## Why This Matters
- Users need scalable challenge for skill development
- Distinct difficulties increase replay value
- Win rates should match difficulty (Easy 80% → Expert 25%)
- Core requirement for v1.0 (REQ-5.1)

### Task 2.5.4: Implement Difficulty-Specific Behavior
**Type**: auto
**Duration**: ~60 min

**Actions**:
1. Update AI decision-making to use difficulty config:
```typescript
function shouldMakeBlunder(difficulty: DifficultyLevel): boolean {
  const config = DIFFICULTY_CONFIGS[difficulty];
  return Math.random() < config.blunderChance;
}

function evaluateMove(move: Move, difficulty: DifficultyLevel): number {
  const config = DIFFICULTY_CONFIGS[difficulty];
  let score = baseEvaluation(move);
  
  // Apply randomness for lower difficulties
  if (Math.random() < config.randomnessFactor) {
    score += (Math.random() - 0.5) * 2; // -1 to +1 modifier
  }
  
  return score;
}
```

2. Add difficulty-specific behaviors:
   - Easy: Sometimes makes obviously bad trades
   - Medium: Occasionally misses optimal plays
   - Hard: Rarely makes mistakes
   - Expert: Always finds best play

3. Test each behavior in isolation

**Verification**:
- Blunders occur at configured rate
- Randomness affects decisions appropriately

### Task 2.5.5: Playtest Each Difficulty
**Type**: checkpoint:human-verify
**Duration**: ~90 min

**What Built**: Tuned difficulty configs

**How to Verify**:
1. Prepare 4 test sessions (one per difficulty)
2. Play 10 games at each difficulty level
3. Record results:
   - Wins/losses
   - Turn count
   - Notable AI decisions (good and bad)
   - Subjective difficulty rating (1-10)

4. Calculate win rates:
   - Easy: Target 70-90%
   - Medium: Target 50-70%
   - Hard: Target 30-50%
   - Expert: Target 15-35%

5. Adjust configs if win rates outside targets
