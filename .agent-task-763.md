## Task

Implement `detectTriggeredAbilities(state, event): TriggeredAbility[]` function per CR 603.2.

## Key Files

- `src/lib/game-state/abilities.ts` - Contains TriggeredAbilityInstance, hasTriggeredAbilities()
- `src/ai/trigger-chain-evaluator.ts` - Existing trigger chain logic
- `src/lib/game-state/types.ts` - GameState types

## Required Implementation

1. Create `detectTriggeredAbilities(state: GameState, event: GameEvent): TriggeredAbilityInstance[]`
2. Implementation must:
   - Scan battlefield for cards with when-clause effects
   - Match event type against trigger conditions (e.g., "when dies" → death event)
   - Return triggered abilities that should fire
   - Order by APNAP order (active player first, then non-active players)

3. Common trigger events to check:
   - DRAW_CARD → triggers like "when you draw a card"
   - DEAL_DAMAGE → triggers like "whenever a creature deals damage"
   - ZONE_CHANGE → triggers like "when enters battlefield", "when dies"
   - ATTACK_DECLARE → triggers like "when attacks"

4. Integrate into game state flow - call after every mutation

## Acceptance Criteria

- detectTriggeredAbilities() function exists and is called after relevant events
- Triggered abilities are placed on stack in correct order
- APNAP order is respected for triggered ability ordering
